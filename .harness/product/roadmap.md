## Shipped (v1.4.2 — 2026-06-10)

- [x] Release packaging fix: app is now built with xcodebuild and ships its SPM resource bundles in Contents/Resources (the v1.4.0/v1.4.1 app artifacts crashed at launch; both releases carry warning notes)

## Shipped (v1.4.0 — 2026-06-10)

- [x] Curled-shell armadillo menu bar icon (template image, generated by `scripts/gen_menubar_icon.py`)

## Shipped (v1.3.0 — 2026-06-10)

Security hardening release from a full project security review:

- [x] Fail-closed app unlock when device authentication is unavailable
- [x] Clipboard copies marked `org.nspasteboard.ConcealedType`; `copy` auto-clear no longer exposes the value in process args
- [x] `add`/`set` accept the value via hidden prompt or stdin to keep secrets out of shell history
- [x] Export salt generation fails loudly on RNG error
- [x] Removed unenforced keychain `.userPresence` flag; trust boundary documented in README
- [x] argon2 dependency pinned to an exact revision
- [x] `SHA256SUMS` published with release artifacts

## Shipped (v1.2.4)

All core features are implemented and distributed via Homebrew.

- [x] Encrypted SQLite vault (AES-256-GCM, vault key in Apple Keychain)
- [x] Touch ID / device-passcode unlock with session-based timeout
- [x] Menu bar app (NSStatusItem + NSPopover, no dock icon)
- [x] Global keyboard shortcut to open popover (configurable, default `⌘⇧Space`)
- [x] Recent secrets list in popover (last 5 copied, persisted in UserDefaults)
- [x] Full CLI: `add`, `set`, `delete`, `get`, `copy`, `list`, `status`, `init`
- [x] Projects: create, list, use, delete, link to directory for auto-resolution
- [x] Environments: per-project profiles, per-environment secret values
- [x] `.env` import (skip-existing by default, `--overwrite` flag)
- [x] Export: encrypted (Argon2id envelope), plain JSON, `.env` format
- [x] Shell injection: `eval $(lokalite shell)` with optional key filter
- [x] Subprocess injection: `lokalite run -- <cmd>` scopes secrets to subprocess only
- [x] MCP server (stdio, JSON-RPC 2.0): read-only by default, `--read-write` flag for writes
- [x] `lokalite install` writes MCP config to `~/.claude.json` automatically
- [x] Three-column settings window: project sidebar, environment + secrets list, secret detail
- [x] Clipboard auto-clear (configurable timeout, default 30s)
- [x] Appearance mode (System / Light / Dark)
- [x] Launch at login
- [x] Homebrew formula (CLI) and cask (app)
- [x] GitHub Actions release pipeline with automated Homebrew PR

## Should-have

These are realistic next priorities given what is already built:

- [ ] **Popover quick-actions refactor** — turn the menu bar popover into a keyboard-first launcher (empty-recents fallback, row hierarchy, Lock in footer, ↓/↑+⏎ copy flow, copy as KEY=value / export / .env, value peek); spec in [popover-refactor.md](popover-refactor.md)
- [ ] **Secret search in CLI** — `lokalite list --search <term>` to filter by name or description from the terminal; the app already supports real-time search
- [ ] **Secret categories / tags** — the schema supports tags (stored as JSON array) but no CLI or app UI exposes them yet; adding a `--tag` flag and tag filter would unlock grouping
- [ ] **Audit log** — local-only, append-only log of access events (read, copy, write) with timestamps; useful for compliance-conscious users
- [ ] **Vault backup / restore workflow** — encrypted export is implemented but there is no guided backup reminder or restore command; a `lokalite backup` and `lokalite restore` command pair would make this a first-class workflow
- [ ] **Multiple vaults** — single vault per machine today; supporting named vaults would let consultants maintain fully separate workspaces per client
- [ ] **Secret versioning** — keep prior encrypted values when a secret is updated so it can be rolled back; schema already supports adding a `version` column to `secret_values`
- [ ] **MCP context awareness** — let the MCP server resolve secrets for the calling agent's working directory automatically (using the project `path` link) without requiring the caller to specify a project name

## Nice-to-have

- [ ] Cross-platform support (Windows, Linux) — extend beyond macOS
